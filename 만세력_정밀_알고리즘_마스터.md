# 만세력 정밀 알고리즘 마스터 가이드

이 문서는 AI가 대한민국 사주 명식(Gan-Zhi)을 산출할 때, 오차 없이 정확한 결과를 도출하기 위한 **정밀 알고리즘 명세서**입니다.
이 로직은 천문학적 계산(Skyfield)과 대한민국 표준시 변천사를 모두 반영한 **검증된 최신 알고리즘**입니다.

## 1. 사주 산출의 핵심 원칙 (Golden Rules)

1.  **반드시 진태양시(True Solar Time)를 사용한다.**
    *   사용자가 입력한 시간은 **행정구역 시간(KST)**입니다. 이를 **경도(Longitude)**와 **균시차(Equation of Time)**를 고려하여 **지역평균태양시(LMT)**로 변환해야 정확한 시주(Hour Pillar)를 얻을 수 있습니다.
2.  **UTC 기준으로 절기(Solar Term)를 계산한다.**
    *   24절기의 시각은 태양의 황경(Apparent Solar Longitude)이 15도 간격으로 변하는 순간입니다. 이는 전 세계 어디서나 동일한 **절대 시각(UTC)**입니다.
3.  **대한민국 표준시 역사를 반영한다.**
    *   과거 127.5도, 135도 등 표준 자오선이 변경되었으며, 서머타임(DST) 시행 기간도 존재합니다. 이를 무시하면 최대 **90분의 오차**가 발생합니다.

---

## 2. 대한민국 표준시 및 서머타임 역사 (Timezone History)

가장 중요한 데이터입니다. 입력된 생년월일에 따라 `UTC Offset`을 정확히 적용해야 합니다.

| 기간 (YYYY.MM.DD) ~ (YYYY.MM.DD) | 표준 자오선 | UTC Offset | 비고 |
| :--- | :--- | :--- | :--- |
| **1908.04.01 ~ 1911.12.31** | 127.5°E | **+08:30** | 대한제국 표준시 |
| **1912.01.01 ~ 1954.03.20** | 135.0°E | **+09:00** | 일제강점기 및 초기 (동경 표준시) |
| **1954.03.21 ~ 1961.08.09** | 127.5°E | **+08:30** | 이승만 정부 변경 표준시 |
| **1961.08.10 ~ 현재** | 135.0°E | **+09:00** | 현행 표준시 (KST) |

### 서머타임(DST) 시행 기간
해당 기간 내에는 시계를 **1시간 앞당겼으므로**, 실제 태양시 계산 시 **-1시간**을 해야 합니다.

*   **1948 ~ 1951년**: 매년 5월 ~ 9월경 시행 (연도별 상이, 대략 5월 첫 일요일 ~ 9월 둘째 일요일)
    *   1948: 06.01 ~ 09.13
    *   1949: 04.03 ~ 09.11
    *   1950: 04.01 ~ 09.11
    *   1951: 05.06 ~ 09.09
*   **1955 ~ 1960년**: 매년 5월 ~ 9월경 시행
    *   1955: 05.05 ~ 09.09
    *   1956: 05.20 ~ 09.30
    *   1957: 05.05 ~ 09.22
    *   1958: 05.04 ~ 09.21
    *   1959: 05.03 ~ 09.20
    *   1960: 05.01 ~ 09.18
*   **1987 ~ 1988년**: 올림픽 기간 시행
    *   1987: 05.10 ~ 10.11
    *   1988: 05.08 ~ 10.09

## 3. 시주(Hour Pillar) 산출 알고리즘 (LMT 계산법)

시주(태어난 시)는 반드시 **진태양시(True Solar Time)**를 기준으로 판단합니다.

### 1단계: UTC 변환
`입력 시간(KST) - (Offset - DST여부)` = **UTC 시간**
*   예: 1988년 5월 20일 13:00 (서울) -> DST 적용(+1h) -> 실제 표준시 12:00 -> UTC = 12:00 - 9h = 03:00 UTC.

### 2단계: LMT(지방평균시) 변환
`LMT = 표준시 + (출생지 경도 - 표준 자오선) * 4분`
*   **서울(127.0°E)**, 표준자오선(135.0°E) 기준
*   차이: -8.0도
*   시간 보정: -8.0 * 4분 = **-32분**
*   결론: 시계가 13:30일 때, 서울의 실제 태양 위치는 12:58입니다. **약 30분 늦게** 자시(23:30~)가 시작되는 이유입니다.

### 3단계: 시주 결정 (표준 자오선 135도 기준 서울 지역)
| 시간 범위 (LMT 기준이 아님, 보정된 KST 기준) | 시주 (Gan-Zhi) | 
| :--- | :--- |
| **23:30 ~ 01:30** | **자시 (Rat)** |
| **01:30 ~ 03:30** | **축시 (Ox)** |
| **03:30 ~ 05:30** | **인시 (Tiger)** |
| **05:30 ~ 07:30** | **묘시 (Rabbit)** |
| ... (2시간 간격) ... | ... |
| **21:30 ~ 23:30** | **해시 (Pig)** |

*   주의: 127.5도 표준시(1954~1961) 시절에는 **정각(00:00, 02:00...)**이 기준이 됩니다.

## 4. 연주(Year)와 월주(Month) 산출 알고리즘

### 연주 (Year Pillar)
*   **기준**: **입춘(Lichun, 황경 315도)** 시각.
*   양력 1월 1일이 아니라, **입춘점(UTC)**을 지났느냐가 기준입니다.
*   예: 2024년 2월 4일 16:27이 입춘이라면, 16:26생은 **2023년(계묘년)**, 16:28생은 **2024년(갑진년)**입니다.

### 월주 (Month Pillar)
*   **기준**: **24절기(Jolgi)** 시각. (입춘, 경칩, 청명, 입하, 망종, 소서, 입추, 백로, 한로, 입동, 대설, 소한)
*   해당 월의 절기 시각을 지났는지 정밀하게 비교해야 합니다.
*   **월건(Month Stem)**: 연간(Year Stem)에 의해 결정됩니다 (진법).
    *   갑기년 -> 병인월 시작
    *   을경년 -> 무인월 시작
    *   병신년 -> 경인월 시작
    *   정임년 -> 임인월 시작
    *   무계년 -> 갑인월 시작

## 5. 대운수(Daewoon Age) 계산법 (Updated)

대운수는 태어난 날로부터 다음 절기(순행) 또는 이전 절기(역행)까지의 **날짜 차이**를 연수로 환산한 값입니다.

### 공식
1.  **방향 결정**:
    *   양남음녀(Yang Male, Yin Female) -> **순행 (Forward)** -> 다음 절기까지 남은 일수 계산.
    *   음남양녀(Yin Male, Yang Female) -> **역행 (Backward)** -> 이전 절기부터 지난 일수 계산.
2.  **시간 차이 계산 (Diff)**:
    *   `Diff = |절기 시각 - 출생 시각|` (분 단위까지 정밀 계산 추천)
3.  **환산 (3일 = 1년)**:
    *   `대운수 = Diff(일) / 3`
4.  **반올림 (사사오입)**:
    *   기존 버림/올림 혼용이 아닌, **정확한 반올림(Round Half Up)**을 사용합니다.
    *   `Final Age = int( (Diff_Days / 3.0) + 0.5 )`
    *   최소값은 1입니다.

## 6. 야자시 / 조자시 (Night Rate / Morning Rat)

*   현대 명리학의 정설을 따릅니다: **일진(Day)의 변경 기준은 23:30(자시 시작)이 아니라 00:00(자정)입니다.**
*   **야자시 (23:30 ~ 00:00)**: 시주는 '자시'를 쓰지만, 일주는 **오늘**의 일진을 그대로 씁니다.
*   **조자시 (00:00 ~ 01:30)**: 시주는 '자시'를 쓰고, 일주는 **내일**의 일진(바뀐 일진)을 씁니다.
*   (일부 학파는 23:30에 일진을 바꾸기도 하므로, 이는 선택 사항이나 본 알고리즘은 **현대적 시각(자정 변경)**을 권장합니다.)
