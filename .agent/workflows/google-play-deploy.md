이미 개발된 앱(AAB 파일)을 구글 플레이 콘솔에 등록하고 출시하는 과정입니다. 코다리 부장이 이 문서를 읽고 설정을 대신 수행합니다.

## 1. 사전 준비물
- [ ] Vercel/Netlify에 배포된 개인정보처리방침 URL (예: https://[app-name].vercel.app/privacy)
- [ ] 앱 아이콘 (512x512, PNG)
- [ ] 그래픽 이미지 (1024x500, PNG)
- [ ] 휴대전화 스크린샷 (최소 2~4장)
- [ ] 출시용 앱 번들 (.aab 파일)

## 2. 앱 설정 항목별 정답지 (코다리 자동입력용)
이 항목들은 구글 심사를 가장 빠르고 안전하게 통과하기 위한 '갓생 세팅'입니다.

- **개인정보처리방침:** `[배포주소]/privacy` 입력
- **앱 액세스 권한:** "특수한 액세스 권한 없이 모든 기능을 이용할 수 있음"
- **광고:** "아니요, 앱에 광고가 포함되어 있지 않습니다."
- **콘텐츠 등급:** 
    - 카테고리: 기타/서비스
    - 모든 질문에 "아니요" (폭력성, 선정성 등 없음) -> '3세 이상' 획득
- **타겟층:** "만 18세 이상" 및 "어린이의 관심을 끌 수 없음(아니오)"
- **데이터 보안:**
    - "수집하거나 공유하는 데이터가 없음" (서버 저장은 수집이 아님)
    - 전송 중 암호화 "예", 데이터 삭제 수단 제공 "예"
- **정부 앱:** "아니요"
- **금융/건강 기능:** "제공하지 않음"
- **카테고리:** "엔터테인먼트" 또는 "라이프스타일"

## 3. 출시 및 게시 절차
1. **국가 및 지역 추가:** [프로덕션] 또는 [비공개 테스트] 메뉴에서 [대한민국] 반드시 추가
2. **테스터 등록:** [테스터] 탭에서 이메일 목록 생성 및 선택
3. **버전 출시:** [새 버전 만들기] -> 라이브러리에서 최신 AAB 파일 추가 -> [버전 검토] -> [출시 시작]
4. **최종 전송:** [게시 개요] 메뉴로 이동하여 [검토를 위해 변경사항 전송] 버튼 클릭
